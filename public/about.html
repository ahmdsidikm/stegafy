01

Metode Steganografi yang Dipakai

Stegafy menggunakan teknik File Appending Steganography — data rahasia dikemas, dienkripsi, lalu ditambahkan di bagian akhir file cover tanpa merusak konten aslinya sama sekali.

💡
Steganografi vs Kriptografi: Kriptografi mengacak data supaya tidak terbaca. Steganografi menyembunyikan keberadaan data itu sendiri. Stegafy menggabungkan keduanya — data disembunyikan sekaligus dienkripsi.

Apa itu File Appending?

Hampir semua format file populer (JPEG, PNG, MP3, MP4) memiliki penanda akhir konten (end-of-file marker). Aplikasi pembaca hanya memproses data sampai penanda itu — byte apapun yang dituliskan setelahnya diabaikan sepenuhnya. Gambar tetap tampil, audio tetap diputar, seolah tidak ada yang berubah.

Stegafy memanfaatkan ini: payload tersembunyi dikemas dan disisipkan tepat setelah penanda EOF file cover.

Kenapa Format Populer Bisa Menyimpan Data Tersembunyi?

🖼️

JPEG / PNG

JPEG berakhir di byte FF D9 (EOI marker). Data setelahnya diabaikan. PNG ditutup oleh chunk IEND berisi CRC — semua setelahnya tidak diproses decoder.

🎵

MP3 / WAV

MP3 dibaca frame per frame berdasarkan sync word. WAV menggunakan header data chunk dengan panjang eksplisit — audio player berhenti di batas tersebut.

🎬

MP4 / Video

Format MP4 (ISO Base Media) memetakan konten via atom/box berindeks. Data di luar atom yang diketahui tidak diproses player.

📄

File Teks

Payload biner ditambahkan setelah teks. Editor teks akan menampilkan karakter aneh di akhir file — sebaiknya jangan buka file teks stego di editor teks biasa.

⚠️
Hati-hati rekompresi platform! Instagram, WhatsApp, dan Telegram (mode Foto) melakukan rekompresi otomatis yang merusak payload. Gunakan Telegram (kirim sebagai File), Google Drive, atau layanan cloud yang menjaga file asli.

Metode Enkripsi

Sebelum ditambahkan ke file cover, payload dienkripsi menggunakan salah satu dari dua metode yang bisa dipilih pengguna.

⚡ XOR Cipher — Mode Standar

XOR (Exclusive OR) adalah operasi bitwise paling fundamental. Setiap byte data di-XOR dengan byte kunci yang diulang secara siklis. Hasilnya sangat cepat dan ringan, cocok untuk browser.

// XOR encrypt — Stegafy function xorEncrypt(data: Uint8Array, key: Uint8Array): Uint8Array { const out = new Uint8Array(data.length); for (let i = 0; i < data.length; i++) { out[i] = data[i] ^ key[i % key.length]; // kunci diulang siklis } return out; } // Dekripsi = operasi yang sama persis (XOR bersifat simetris)
⚠️
Keterbatasan XOR: Rentan terhadap known-plaintext attack dan analisis frekuensi jika kunci pendek atau berulang secara prediktabel. Gunakan untuk data tidak sensitif atau saat kecepatan lebih penting daripada keamanan kriptografis.

🛡️ AES-256-GCM — Mode Pro

Advanced Encryption Standard dengan kunci 256-bit dalam mode Galois/Counter (GCM). Standar enkripsi yang digunakan militer AS, perbankan global, dan infrastruktur internet (TLS/HTTPS).

🔐

AES-256 — Block Cipher

Data diproses dalam blok 128-bit menggunakan kunci 256-bit (2²⁵⁶ kemungkinan kunci). Tidak bisa di-brute-force dengan teknologi yang ada — bahkan komputer kuantum pun butuh triliunan tahun.

✅

GCM — Authenticated Encryption

GCM menghasilkan authentication tag 128-bit. Jika ciphertext dirusak atau dimodifikasi oleh pihak manapun, dekripsi langsung gagal dengan error — data palsu terdeteksi otomatis.

🧮 Argon2id — Key Derivation Function

Password pengguna tidak langsung dipakai sebagai kunci AES. Argon2id mengubahnya menjadi kunci 256-bit melalui proses yang sengaja dibuat mahal secara komputasi dan memori, sehingga serangan brute-force menjadi tidak praktis secara ekonomis.

🧠

Memory: 64 MB

Setiap percobaan password butuh 64 MB RAM. GPU dengan ribuan core pun dibatasi berapa banyak percobaan bisa berjalan paralel.

🔁

3 Iterasi

Password diproses tiga kali. Menambah waktu komputasi tanpa mengurangi kebutuhan memori. Total waktu ~1–3 detik.

⚙️

Parallelism: 4

Memanfaatkan 4 thread untuk efisiensi sambil membatasi paralelisme serangan.

⚡

Kapan pakai Mode Standar?

Kamu hanya ingin menyembunyikan file dari pandangan umum, bukan dari pihak yang aktif mencuriga. Data tidak terlalu sensitif. Kecepatan lebih penting.

🛡️

Kapan pakai Mode Pro?

Data benar-benar sensitif: dokumen pribadi, keuangan, hukum. Atau kamu ingin Face Lock dan kunci yang tidak bisa ditebak manusia.

⚠️
XOR bukan enkripsi yang kuat. Dengan password pendek dan data yang diketahui sebagian (known-plaintext attack), XOR bisa dipecahkan. Gunakan Mode Pro untuk data sensitif.

Face Lock — Keamanan Biometrik

Eksklusif Mode Pro. File hanya bisa didekripsi jika wajah yang memindai cocok dengan wajah yang didaftarkan saat enkripsi. Semua pemrosesan terjadi 100% di browser — tidak ada data wajah yang pernah dikirim ke server.

📸

Saat Enroll (Enkripsi)

Wajah dipindai, 128 nilai float32 diekstrak menjadi vektor fitur, lalu disimpan terenkripsi di dalam file stego. Vektor ini tidak bisa dibalikkan menjadi foto wajahmu.

✅

Saat Verify (Dekripsi)

Wajah dipindai ulang, vektor baru dihitung, lalu jarak Euclidean antara keduanya dihitung. Jika di bawah threshold 0.5, akses diberikan.

// Euclidean distance antara dua descriptor wajah function faceDescriptorDistance(a: Float32Array, b: Float32Array) { let sum = 0; for (let i = 0; i < a.length; i++) { sum += (a[i] - b[i]) ** 2; } return Math.sqrt(sum); } // Threshold default: 0.5 // distance < 0.5 → wajah cocok → akses diberikan const FACE_MATCH_THRESHOLD = 0.5; const isMatch = faceDescriptorDistance(live, stored) < FACE_MATCH_THRESHOLD;
⚠️
Selalu simpan password sebagai backup! Perubahan penampilan drastis (kacamata baru, rambut berbeda, pencahayaan sangat buruk) bisa membuat Face Lock gagal. Password tetap bisa digunakan sebagai akses alternatif.

Alur Penggunaan

🔒 Menyembunyikan File

1

Pilih File Cover

Upload file wadah — gambar (JPEG/PNG/GIF/WebP), audio (MP3/WAV), video (MP4), atau teks. File ini tetap bisa dibuka normal setelah proses.

2

Tambah File Rahasia

Satu atau lebih file apapun. Tambahkan komentar per-file jika perlu. Nama file bisa diubah untuk menyamarkan nama aslinya.

3

Pilih Metode & Password

Mode Standar (XOR) — cepat, password opsional. Mode Pro (AES-256) — keamanan penuh, password manual atau generated key 50 karakter kriptografis.

4

Scan wajahmu untuk menambah lapisan keamanan biometrik. Vektor wajah akan dienkripsi bersama payload.

5

Unduh File Stego

Klik "Sembunyikan & Enkripsi". Jika pakai generated key, unduh key.sty dan simpan di tempat aman yang berbeda dari file stego.

06

Model Keamanan

🖥️

100% Client-Side

Semua kriptografi berjalan di browser via Web Crypto API. Tidak ada file, password, atau kunci yang pernah dikirim ke server mana pun.

🧹

Memory Wiping

Setelah operasi selesai, password dihapus dari memori JavaScript menggunakan secureWipeString() untuk meminimalkan jejak di RAM.

🎲

Salt Unik per File

Setiap enkripsi menggunakan salt 16-byte baru dari crypto.getRandomValues(). Mencegah serangan rainbow table dan pre-computation.

🔏

IV Acak per Operasi

AES-GCM menggunakan IV 12-byte baru setiap enkripsi. Ciphertext selalu berbeda meski plaintext dan kunci sama — mencegah known-ciphertext analysis.

⚠️
Batas Perlindungan: Stegafy tidak melindungi dari steganalisis aktif — jika seseorang sudah curiga bahwa file menyimpan data tersembunyi dan melakukan analisis forensik, keberadaan payload bisa terdeteksi (meski isinya tetap terenkripsi kuat). Keamanan konten sepenuhnya bergantung pada kekuatan password.
💡
Tips Password Kuat: Minimal 16 karakter, campuran huruf besar/kecil, angka, dan simbol. Atau gunakan fitur Generate Key yang membuat kunci 50 karakter kriptografis acak — entropi jauh lebih tinggi dari password manual mana pun.

FAQ

Apakah file cover rusak setelah disematkan data?+
Tidak. File cover tetap bisa dibuka dan berfungsi normal — gambar tetap tampil, audio tetap diputar. Data tersembunyi hanya ditambahkan di luar batas konten utama file, yang diabaikan oleh aplikasi pembaca biasa.
Berapa banyak file yang bisa disembunyikan?+
Tidak ada batasan jumlah. File stego akan berukuran file cover + total semua file rahasia + overhead kecil (header, salt, IV). Satu-satunya batas praktis adalah memori browser dan kenyamanan pengguna.
Apakah data bisa dipulihkan jika password hilang?+
Tidak. AES-256 yang diimplementasikan dengan benar tidak memiliki backdoor. Jika password hilang dan tidak ada backup file key, data tidak dapat dipulihkan. Selalu simpan password atau file key.sty di tempat yang aman dan terpisah.
Kenapa file stego rusak setelah dikirim lewat WhatsApp/Instagram?+
Platform tersebut melakukan rekompresi otomatis pada gambar dan video. Ini mengubah byte-byte di luar konten visual dan merusak payload. Gunakan Telegram (kirim sebagai File), Google Drive, Dropbox, atau email dengan attachment untuk menjaga integritas file.
Apakah data wajah dikirim ke server?+
Tidak sama sekali. Model AI face-api.js berjalan sepenuhnya di browser via WebGL. Vektor wajah 128-float32 hanya tersimpan di dalam file stego itu sendiri — tidak pernah meninggalkan perangkatmu.
Apa itu file .sty dan mengapa lebih aman dari password biasa?+
File .sty berisi kunci 50 karakter acak yang di-generate via crypto.getRandomValues() — sumber keacakan kriptografis browser. Entropi-nya jauh lebih tinggi dari password yang diingat manusia. Simpan file ini terpisah dari file stego karena hilangnya file key = data tidak bisa dipulihkan.
Bisakah orang lain tahu bahwa sebuah file adalah file stego?+
Secara visual tidak terlihat perbedaan. Namun dengan analisis forensik (steganalisis) seperti membandingkan ukuran file yang tidak biasa atau mencari magic header Stegafy, keberadaan payload bisa terdeteksi. Isi payload tetap terenkripsi dan tidak bisa dibaca tanpa kunci yang benar.
